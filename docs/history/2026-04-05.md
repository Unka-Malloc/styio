  - 未显式类型参数也保有默认 `TypeAST`。
- hash iterator 定义恢复：
  - legacy/new `# ... >> ...` 分支恢复接管（保留单参数约束）。
- 回归更新：
  - `StyioParserEngine.LegacyAndNewMatchOnHashIteratorDefinition` 改为成功路径断言（exit 0 + 双引擎输出一致）；

  - `StyioSecurityNewParserStmt.MatchesLegacyOnFunctionDefSubsetSamples` 重新纳入 hash iterator 定义样例。
- 定向回归结果：
  - `StyioSecurityNewParser(Expr|Stmt|Shadow).*` 全绿；
  - `StyioParserEngine.*` 全绿；
  - `# iter_only(x) >> (n) => >_(n)` 样例在 legacy/new 下均退出码 `0`。
- ADR 落地：`docs/adr/ADR-0053-param-ctor-null-type-and-hash-iterator-restore.md`。

## Checkpoint（E.7 稳定性收口：Hash Iterator `?=` Forward Chain Guard）

### 当前状态

- 收敛一个“非崩溃但语义静默错误”路径：
  - `# f(x) >> (n) ?= 2 => ...` 在恢复后会被解析但行为不符合预期（静默空输出）。
- 在 `parse_iterator_with_forward` 增加 guard：
  - 首子句为 `CheckEqual/Cases` 时，稳定拒绝；
  - 多子句 forward chain（`size > 1`）时，稳定拒绝。
- 回归补齐：
  - `StyioParserEngine.HashIteratorMatchForwardChainReturnsParseError`（双引擎 JSONL：`ParseError` + 退出码 `3`）。
- 兼容性验证：
  - `StyioSecurityNewParser(Expr|Stmt|Shadow).*` 全绿；
  - `StyioParserEngine.*` 全绿；
  - shadow gate：M1 `40/40`，M2 `20/20`。
- ADR 落地：`docs/adr/ADR-0054-hash-iterator-match-forward-chain-guard.md`。

### 下一步

1. E.7 后续：为 iterator `?=` forward 组合补独立语义设计，再解除 guard。
2. D 阶段联动：将该失败样本加入 soak/回归模板，防止 silent behavior 回归。
3. F.1：统一这类“暂不支持组合语法”的诊断文案模板。

## Checkpoint（E.7 风险收口：Hash `>>` Iterator 定义 Crash Guard）

### 当前状态

- legacy/new 在 `# f(x) >> ...` 定义路径统一改为稳定拒绝：
  - 抛出 `StyioNotImplemented("hash iterator definitions are temporarily disabled (runtime crash guard)")`；
  - CLI 侧统一映射为 `ParseError`（退出码 `3`），不再段错误退出。
- 安全回归更新：
  - 删除“hash iterator 定义应成功”等价样例；
  - 新增 `RejectsHashIteratorFunctionDefWithStableError`，断言 legacy/new 都抛 `StyioBaseException`。
- parser-engine 回归新增：
  - `HashIteratorDefinitionReturnsParseErrorWithoutCrash`；
  - 双引擎在 `--error-format=jsonl` 下都返回退出码 `3`，且包含 `"category":"ParseError"`。
- shadow gate 结果保持：
  - M1 `40/40`；
  - M2 `20/20`。
- ADR 落地：`docs/adr/ADR-0052-hash-iterator-crash-guard.md`。

### 下一步

1. E.7 后续：将 `# ... >> ...` 的恢复实现拆成独立子切片（parser 语义保持守卫，优先查 runtime 崩溃根因）。
2. D 阶段联动：为 iterator runtime 增加 soak 压力与内存守卫回归，避免“修复即回归”。
